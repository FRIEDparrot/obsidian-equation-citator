## 2. 自动编号技巧
### (1) 从标题列表中忽略某个标题
这是你在撰写论文时常见的情况，**第一章之前有几个章节**，比如**摘要**或**前言**。如果你不想让**摘要**和正文部分共享编号，可以**使用 HTML 格式的标题**。

```html
<h1> 摘要 </h1>
```

上述标题将从自动编号的标题列表中被忽略。你可以检查 `设置 > 基础 > 无标题时自动编号前缀`，根据需要设置（例如 `A`），则结果将如下所示：
```
<h1> 摘要 </h1>
$$... \tag{A1}$$
# 第一章
$$... \tag{1.1}$$
```

### (2) 使用框选公式工作流程
对于有大量推导的用户，他们可能会使用 `仅自动编号带标签的公式` 和 `更多选项 > 筛选器 > 筛选带标签的项`。但这仍然需要手动添加标签，因此我在这里建议**框选公式工作流程**：

你可以使用 `框选当前公式` 为那些**重要公式添加 `\boxed{}`**。然后在公式面板中，使用 `更多选项 > 筛选器 > 筛选框选公式` 来整理它们。

本插件也支持在 **Typst 模式**下添加框选，默认使用 `"boxed"`，与 `Typst Mate` 一致（参见 [issue #140](https://github.com/FRIEDparrot/obsidian-equation-citator/issues/140)）中的 `Typst 框选符号`。

## 2. 图表和表格引用技巧
### (1) 更轻松地编辑标注内的表格
在标注中编辑表格可能很别扭，因为每行都以 `>` 开头。以下是一个简化此过程的工作流程（另见 [feature #7](https://github.com/FRIEDparrot/obsidian-equation-citator/issues/7)）：

1. 暂时移除引用标记：
   - 使用 `Ctrl + Shift` + 垂直拖动（放置多个光标）。
   - 删除每行表格前面的 `>`。
2. 正常编辑表格。
3. 将表格重新包装到标注中：
   - 插入标注（例如按 `Ctrl + Q`）以重新为所有行添加 `>`。
   - 确保标注标题行（例如 `> [!note]`）正确定位在表格上方。
4. 如果相邻块被分割，将它们合并。

### (2) 更好的标注引用样式

## 3. 将公式和脚注从一个文件移动到另一个文件
当将某个章节（包括公式和脚注）迁移到更大的文档中，同时保持编号整洁时，你应该遵循以下步骤（虽然有些繁琐）。
#### 步骤 1. 移动前更新脚注引用

如果你的源文件有如下脚注定义：
```
[^2]: 引用 [[测试文章]] 的某些注释
```
而目标文件已经使用了直到 `[^5]` 的脚注，则：
- 将 `[^2]` → 重命名为 `[^6]`
- 更新所有与 `2^{…}` 关联的行内引用模式（如果这是你的嵌套跨文件引用风格）。

高效搜索：
- 使用 `Ctrl + H`，搜索词：`2^{`，或搜索引用格式 `\ref{eq:`
- 仔细替换为：`6^{`，手动更新所有引用。

> [!warning]
> 不要一次性全部替换，像 `2^{100}`（指数表达式）这样的模式可能会意外匹配你的引用格式。

#### 步骤 2. 为源文件中的公式应用前缀

1. 前往：`设置 > 分类 > 自动编号 > 自动编号全局前缀`
2. 设置一个前缀（例如 `M`）
3. 在源文件中运行自动编号。

之后，公式变为：
- `M1.1`、`M1.2`、`M1.3`、…

这样可以区分迁移的公式和目的地中原有的公式。

#### 步骤 3. 移动内容

复制或追加：
- 所有内容和带前缀的公式
- 更新后的脚注

将它们粘贴到目标文件的所需章节中。

#### 步骤 4. 更新反向链接

打开原始（现在已空）文件的反向链接面板，如果需要（尤其是在其他笔记链接到旧章节的情况下），更新引用指向合并后的目标文件。

#### 步骤 5. 重新运行自动编号并清理

在目标文件中：
1. 重新运行自动编号（如果集成需要全局重新计算编号）。
2. 验证所有引用仍然可以解析。
3. 如果原始源文件已完全迁移，则将其删除。
